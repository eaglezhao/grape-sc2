The purposes of this document

This document will try to explain what Grape is meant for, what is supported, what is intended, what is allowed and what is not, and will explain the official code style of Grape.
For official resources, samples, spells, etc., this document should be complied with as much as possible. For unofficial resources, using own coding style is allowed but not encourages as the stdlib uses the official code style.

The purpose of Grape

What Grape is meant for

Basically, Grape is meant for fixing the issues that are currently present with the StarCraft 2 native scripting language by the name of Galaxy. Galaxy is a C-like language, meaning that it is a procedural language. When working on larger projects, this will become a problem for maintainability and overview of code. Grape tries to fix this by implementing forced object orientation, meaning that all code has to be contained in a class.
There are also problems with the Galaxy syntax checker. This means that it will sometimes report issues that don't exist, or won't report existing issues. This problem is solved by Grape being both open source and actively developed.

Grape also follows a very minimalistic, simple and readable syntax that is supposed to feel more like a scripting language. It is inspired by Lua and Ruby.
class mod inherits mod_base
    override void main()
        game.display("hello")
    end
end
The above code will display the string "hello" on the screen when the map or mod has been loaded.
